CH5: Objects

  • RegExp
  • For example, say you were looking for any word ending in 'ing', you could use the regular expression /[a-zA-Z]+ing$/.
  • const pattern = /[a-zA-Z]+ing$/;
  • pattern.test('joke'); << false
  • pattern.test('joking'); << true

  • Regular Expression Properties?
  • g sets the global property to true
  • i sets the ignoreCase property to true

  • Special Characters
  • . matches any character, except line breaks
  • \w matches any word character, and is equivalent to [A-Za-z0-9_]
  • \W matches any non-word character, and is equivalent to [\^A-Za-z0-9_
  • \d matches any digit character, and is equivalent to [0-9]
  • \D matches any non-digit character, and is equivalent to [^0-9]
  • \s matches any whitespace character, and is equivalent to [ \t\r\n\f]
  • \S matches any non-whitespace character, and is equivalent to [^ \t\r\n\f]

CH7: Events

Every time a user interacts with a web page, such as clicking on a link, pressing a key, or moving a mouse, an event occurs that our program can detect and then respond to.

  • addEventListener()
  • document.body.addEventListener('click',doSomething);
  • addEventListener('click', () => alert('You Clicked!'));

  • If you need to implement gestures, it's probably a good idea to use a library such as Hammer.JS or zingtouch that makes events such as swipe, pinch and rotate easy to implement.
